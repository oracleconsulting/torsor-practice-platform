1. **Old Database** ‚Üí `invitations.assessment_data` (JSONB) contained skill assessments with old UUIDs

3. **Assessment data migrated** by matching `skill_name` instead of `skill_id`
4. **12 phantom skills** (existed in new schema but not in old assessments) were deactivated
5. **Service line definitions updated** to use the 111 active, assessed skills
